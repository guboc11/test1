# 개발 메모

## 대용량 데이터 내보내기 (미구현)

### 배경
- 시뮬레이션 결과를 사용자가 직접 다운로드해서 신뢰성을 확인할 수 있도록 하고 싶음
- 대상: 일반인 + 데이터 분석가/개발자 혼재
- 목표 데이터 규모: 최대 4400만 개 (실제 유권자 수 기준)

### 각 행 데이터 구조 (안)
개인별 특성 몇 가지로 구성:
- 사람 ID
- 성별
- 나이
- 사는 지역
- 정당 성향
- 기타 특성 (추후 확정)

### 결정된 방향

**포맷: Parquet**
- 바이너리 → CSV 대비 용량 훨씬 작음
- 스키마가 파일 내에 내장 → 별도 스키마 파일 불필요
- 일반인: 브라우저 기반 뷰어로 설치 없이 바로 열람 가능
- 분석가/개발자: pandas, R, DuckDB, Spark 등 네이티브 지원

**일반인용 Parquet 뷰어 (브라우저, 설치 불필요)**
- https://parquetreader.com — SQL 쿼리, CSV 변환 지원
- https://viewparquet.com — 빠른 뷰어
- https://www.chatdb.ai/tools/parquet-reader — WASM 기반, 500MB까지 무난

**다운로드 옵션 두 가지로 분리**
| 대상 | 형식 | 내용 |
|------|------|------|
| 일반인 | CSV | 요약 집계 또는 샘플 (100만 행 이하) |
| 분석가 | Parquet | 전체 원본 데이터 |

### 기술 스택 방향

```
Rust/Go → WASM   (계산: 4400만 개 데이터 생성, JS 대비 10~50배 빠름)
Web Worker       (백그라운드 실행, 메인 스레드 블로킹 방지)
File System Access API  (스트리밍으로 디스크에 직접 저장, 메모리 문제 우회)
```

- Rust/Go 모두 Parquet 라이브러리 잘 갖춰져 있음
- File System Access API: Chrome/Edge 지원, Safari 미지원

### 구현 순서 (예정)
1. UI 먼저 확정
2. 소규모 데이터로 흐름 검증 (JS로 프로토타입)
3. WASM 붙여서 대용량 처리
