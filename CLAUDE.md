# Law of Large Numbers — 대수의 법칙 시뮬레이션

## 프로젝트 목적

대수의 법칙을 직관적으로 납득하지 못하는 사람들을 위해 **실제 시뮬레이션**을 만든다.

주제: 대한민국 선거에서 **사전 투표**와 **본 투표** 결과가 비슷하게 수렴함을 시각적으로 보여주는 것.

## 시뮬레이션 개요

전체 인구 중 일정 비율이 특정 정당을 지지할 때, 사전 투표를 하든 본 투표를 하든 **충분히 큰 표본**에서는 전체 지지율에 수렴한다는 것을 애니메이션/인터랙티브 시각화로 보여준다.

## 파라미터

| 파라미터 | 설명 |
|---|---|
| 정당 지지율 | 전체 인구 중 특정 정당을 지지하는 비율 (%) |
| 전체 인구 수 | 시뮬레이션 대상 인구 |
| 투표 참여 확률 | 전체 인구 중 실제로 투표할 확률 |
| 사전 투표 확률 | 투표자 중 사전 투표를 선택할 확률 |
| 본 투표 확률 | 투표자 중 본 투표를 선택할 확률 |
| 지지 정당 변경 확률 | 사전 투표 후 며칠 사이 다른 정당으로 마음이 바뀔 확률 |

## 기술 스택

- 패키지 매니저: **pnpm** (npm, yarn 사용 금지)
- 프레임워크: React + TypeScript + Vite
- 앱 경로: `app/`

## 핵심 규칙

> **절대로 멋대로 해석하거나, 넘겨짚거나, 가정하지 않는다.**
> 사용자가 한 말에만 집중한다. 말하지 않은 것을 임의로 채워 넣지 않는다.
> 불명확한 부분이 있으면 행동하기 전에 반드시 먼저 질문한다.
> 이를 어기는 것은 가장 심각한 규칙 위반이다.

> **대화 중에는 코드 작성 욕구를 완전히 내려놓는다.**
> 사용자는 충분한 대화를 통해 스스로 구현 지시를 내린다.
> 대화가 끝나지 않았는데 "구현할까요?" 라고 묻거나, 코드를 작성하거나, 파일을 수정하는 행위는 엄격히 금지된다.
> 사용자가 명확히 "구현해줘", "코드 작성해줘", "해줘" 라고 말할 때까지 코드에 손대지 않는다.
> 코드 작성이 목적이 아니라, 사용자의 말을 끝까지 듣는 것이 최우선이다.

> **구현(코드 작성/파일 수정)을 시작하기 전에 반드시 먼저 말로 알린다.**
> "이제 구현하겠습니다" 또는 구현 계획을 텍스트로 먼저 출력한 뒤 도구를 사용한다.
> 대화 중에 갑자기 도구 호출로 구현을 시작하지 않는다.

> **사용자가 명시적으로 "코드를 생성해 줘"라고 말하기 전까지는 절대 코드를 생성하지 않는다.**
> 코드 작성 전에 반드시 "이대로 코드를 생성할까요?" 라고 먼저 물어본다.

> **라이브러리/기능 구현 전, 반드시 기존 라이브러리를 먼저 탐색한다.**
> 어떤 기능이 필요할 때 직접 구현하기 전에 해당 기능을 제공하는 라이브러리가 있는지 먼저 찾아보고,
> 찾은 옵션들을 사용자에게 설명한 뒤 무엇을 쓸지 결정을 맡긴다.
> 적절한 라이브러리가 없을 경우에도 직접 구현하기 전에 반드시 사용자의 허락을 구한다.
